Prompting in practice

Clear prompts beat clever prompts.

Treat the model like a junior analyst who needs structured guidance: role, context, task, output format, limits, and assumptions.

Core formula

Role + Context + Task + Format + Limits + Assumptions

If a prompt is vague, the answer usually gets vague too. Add just enough structure to make the task reviewable.

# Smart Prompt Maker

## Your Role
You are an expert prompt engineer. Your job is to help the user craft a high-quality, effective prompt they can use with any AI tool. You do this through a structured two-phase process: first gathering information, then building the prompt.

---

## Phase 1 — Understand the Task

Ask the user to describe what they want to accomplish. Once they respond, generate 5 to 7 focused clarifying questions tailored specifically to their task. Do not use generic or one-size-fits-all questions. Each question should target a distinct dimension of the task such as audience, goal, tone, format, constraints, context, or success criteria.

Present your questions as a numbered list. Wait for the user to answer before proceeding.

**Rules for your questions:**
- Make every question specific to what the user described
- Cover different angles — do not ask two questions about the same thing
- Keep questions concise and easy to answer
- If useful, include a brief example of the kind of answer you are looking for

---

## Phase 2 — Build the Prompt

Once the user has answered at least half of your questions, synthesize everything into a single, well-structured prompt they can copy and use directly with any AI tool.

**The final prompt must include, where relevant:**
- A clear role or persona for the AI
- Specific instructions tied to the user's task
- Background context and constraints
- Target audience and tone guidance
- Expected output format and length
- Any special requirements or edge cases the user mentioned

**Rules for the final prompt:**
- Write it as if addressing the AI that will execute the task, not the user
- Use plain, direct language that works across all major AI models
- Do not include meta-commentary or explanations inside the prompt itself
- Present it in a clearly labeled, copy-ready block

---

## How to Begin

Start by saying:

"I'll help you build a great prompt. What do you want to accomplish? Describe your task in as much detail as you like — the more context you give me, the better my questions will be."

Starter ideas

Ideas to feed into the Smart Prompt Maker

Triage

Build an alert triage prompt

I want to create a prompt for alert triage.
Context: [alert type], [tool/source], [sanitized fields available].
Goal: help an analyst decide [escalate / close / investigate further].
Output needed: [brief summary], [risk], [evidence gaps], [next steps].
Tone: concise and operational.
Constraints: do not make unsupported conclusions; clearly state uncertainty.
Investigation

Build an IOC correlation prompt

I want to create a prompt for correlating investigation evidence.
Context: [sanitized indicators], [timeline notes], [related alerts or tickets].
Goal: identify relationships, likely themes, and missing evidence.
Output needed: [grouped findings], [confidence level], [follow-up checks].
Tone: analytical but cautious.
Constraints: separate confirmed evidence from hypotheses.
Detection

Build a query helper prompt

I want to create a prompt for improving a security query.
Context: [query language], [data source], [investigation goal].
Goal: explain what the query does and suggest improvements.
Output needed: [plain-language explanation], [blind spots], [cleaner query idea].
Tone: practical and easy to understand.
Constraints: preserve the original detection intent and call out assumptions.
Vulnerability

Build a prioritization prompt

I want to create a prompt for vulnerability prioritization.
Context: [asset exposure], [service role], [known weakness], [available severity data].
Goal: decide what deserves attention first.
Output needed: [priority level], [reasoning], [questions to validate].
Tone: risk-focused and concise.
Constraints: do not rely on severity alone; include exposure and likely attacker interest.
Reporting

Build a handoff prompt

I want to create a prompt for SOC handoff notes.
Context: [sanitized investigation notes], [current status], [known open questions].
Goal: make the handoff clear for the next analyst or shift.
Output needed: [current understanding], [evidence], [risks], [next actions].
Tone: professional and brief.
Constraints: avoid speculation unless clearly labeled.
Learning

Build an intel digest prompt

I want to create a prompt for digesting threat intelligence.
Context: [report type], [threat actor or campaign], [relevant technologies].
Goal: convert the content into SOC action points.
Output needed: [behaviors to watch], [detection ideas], [relevance], [gaps].
Tone: concise and operational.
Constraints: distinguish reported facts from recommended actions.

Quality control

Prompt habits that improve results

Ask for assumptions and uncertainty

This makes it easier to spot weak reasoning and stops the model from sounding more certain than it should.

Define the output shape before the answer starts

Bullet points, tables, short summaries, and prioritized lists are easier to review than free-form text.

Keep context relevant, not maximal

Extra text can dilute the task. Give the model what it needs, then stop.

Use a second pass for refinement

First ask for structure, then ask for tuning. That usually works better than trying to get a perfect answer in one shot.

Avoid this

Common prompt anti-patterns

  • “Analyze this” with no task or output format.
  • Dumping too much unsorted context into one request.
  • Asking for final decisions without verification.
  • Forgetting to sanitize before pasting.
