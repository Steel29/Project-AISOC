Safety and governance

Useful outputs start with disciplined inputs.

AI can save time, but security teams still own sanitization, validation, and the decision to act.

Generally safe to share

Sanitized operational context

  • Redacted alert summaries.
  • Generic log patterns or query goals.
  • Abstracted incident narratives.
  • Process questions and writing tasks.

Do not paste casually

High-risk content

  • Secrets, tokens, passwords, or private keys.
  • Unnecessary personal or customer data.
  • Confidential internal information without approval.
  • Raw content when a sanitized version would do the job.

Practical helper

Sanitizer Algorithm

Regex sanitization is helpful, but it does not replace human review or company policy. Always review the output before sharing it with an AI tool.

01

Minimize

Start with only the log lines, notes, or fields required for the task.

02

Detect

Look for common sensitive patterns such as emails, secrets, IPs, paths, and hostnames.

03

Replace

Swap values for typed placeholders like [REDACTED_EMAIL_1].

04

Preserve shape

Keep structure, timestamps, and analyst notes readable for follow-up analysis.

05

Review

Check the sanitized output manually before pasting it into any AI assistant.

Switch

Handle IOCs

Redact public IOCs by default, or use -KeepPublicIocs when safe and necessary.

PowerShell utility

Copy the SOC text sanitizer

Save this as tools/Sanitize-SocText.ps1, or use the script already included in this repository.

Example usage

Sanitize a file and write a clean copy:

.\tools\Sanitize-SocText.ps1 -InputPath .\notes.txt -OutputPath .\notes.sanitized.txt

Sanitize pipeline input:

Get-Content .\notes.txt -Raw | .\tools\Sanitize-SocText.ps1

Preserve public IOCs when needed:

.\tools\Sanitize-SocText.ps1 -InputPath .\notes.txt -KeepPublicIocs

Placeholder examples

  • analyst@example.com becomes [REDACTED_EMAIL_1]
  • Bearer abc123... becomes [REDACTED_AUTH_HEADER_1]
  • 10.10.4.5 becomes [REDACTED_PRIVATE_IP_1]
  • https://example.com/path becomes [REDACTED_URL_1]

Validation is part of the workflow

Validate factual claims

If the model says an event happened, a query means something specific, or a conclusion is likely, check the evidence directly.

Watch for confident-sounding mistakes

Polished writing can hide incorrect reasoning. Treat the style and the substance as separate things.

Separate drafting from deciding

AI can draft the narrative or suggest next steps. Analysts still decide what is true and what should happen next.
